# Watercolor images — handoff to Lovable

The site currently uses a hand-drawn SVG (`components/farm-scene.tsx`) on
all auth pages. The plan is to replace the big bleed illustrations with
warm watercolor JPEGs generated by Gemini, while keeping the small icons
(`Mark`, `Sun`, `Wheat`, `Jar`, `Leaf`, `Barn`) as SVG so they stay
tintable with our color tokens.

This doc gives you everything to do the swap in one pass.

---

## Where the images go

Drop generated files into `public/watercolors/` at the repo root. The
file names are wired up already — once the files exist and you set
`NEXT_PUBLIC_USE_WATERCOLORS=true`, the auth pages swap automatically.

```
public/
└── watercolors/
    ├── dawn-pasture.jpg    ← /farmer/* auth pages
    ├── dusk-table.jpg      ← /come-in (member sign-in)
    ├── field-rows.jpg      ← future: landing hero
    └── hands-and-soil.jpg  ← future: manifesto / about
```

To enable, set the env var:

```bash
# In Lovable's env config or .env.local
NEXT_PUBLIC_USE_WATERCOLORS=true
```

The `<WatercolorScene>` component in `components/watercolor-scene.tsx`
handles the swap. If a file is missing it falls back to the SVG so the
page never breaks.

## The four images we need (priority order)

### 1. `dawn-pasture.jpg` — operator sign-in, sign-up, password reset

**Use:** Left half of all four `/farmer/*` auth pages. Portrait, deeply
cropped, full-bleed under a `bg-soil` background.

**Aspect & size:**
- Aspect ratio: **2:3 portrait** (renders at 800 × 1200 in the SVG fallback)
- Recommended generation: **1024 × 1536** or larger; we downscale
- Format: JPEG, 80–85% quality, ~250–400 KB final

**Gemini prompt:**

> A soft watercolor painting of a small American family farm at dawn.
> Rolling green pastures dotted with two or three jersey cows, a wooden
> split-rail fence in the middle ground, a low red barn at the
> horizon to the right, a single oak tree to the left. The sky is warm
> peach and apricot fading to a pale buttery yellow at the horizon —
> the moment just after sunrise. The whole image has a hand-painted
> editorial quality, like an illustration from a 1970s back-to-the-land
> magazine or a Wendell Berry book cover. Loose washes, visible brush
> strokes, gentle granulation in the wet-on-wet areas. Warm earth-tone
> palette: peach, cream, soft moss green, brick red. No people in the
> frame. No text. No logos. Painterly, not photographic.

## Voice / style notes for the prompts above

Communicare's brand voice is editorial — Catholic Worker, Rerum Novarum,
Wendell Berry. The images should feel like illustrations from a small
magazine or a back-to-the-land essay collection. Not stock-photo glossy,
not children's-book whimsical, not AI-shiny.

Hit these notes in every prompt:

- **Loose washes, visible brush strokes, granulating pigment** — these
  are the watercolor signifiers Gemini knows.
- **Warm earth tones** — peach, amber, brick, wheat gold, moss green.
  Match the site palette (`tailwind.config.ts`).
- **No people in frame** (or painted so loosely they're a suggestion).
  We're not selling lifestyle; we're selling the work.
- **No text, no logos, no readable labels.** Anything legible will look
  AI-generated.
- **Reference period:** 1970s editorial illustration is the sweet spot —
  Saveur, Audubon plates, Whole Earth Catalog. Avoid "modern watercolor
  Instagram art."

## What stays SVG (do not replace)

These are functional UI elements — they need to tint with color tokens,
scale crisp at any size, and stay light in the bundle. Keep them as is:

| Component                                 | Why                                                                                   |
|-------------------------------------------|---------------------------------------------------------------------------------------|
| `components/mark.tsx` — Mark logo + icons | Brand mark + the Sun/Wheat/Jar/Leaf/Barn icons. They tint inline via `className`.    |
| `components/screenshots.tsx`              | In-app UI mockups for the landing page. Pure React DOM, not really SVG anyway.        |
| `components/share/pickup-ticket.tsx`      | The pickup-ticket SVG glyphs. Need to tint based on share kind.                       |
| Topographic atlas on `/find`              | Functional — farm pins are mathematically projected onto the SVG canvas.              |
| `app/farmer/share-cards/page.tsx` SVGs    | The printable share-card mockups. SVG so they print at any size.                      |
| `app/icon.svg`                            | The browser favicon. Has to be SVG.                                                   |
| `components/farm-scene.tsx`               | Keep as the fallback — referenced by `<WatercolorScene>` when images aren't present.  |

## Verifying the swap

After dropping images into `public/watercolors/` and setting the env
var, restart dev (or rebuild) and visit:

- `/come-in` — should show `dusk-table.jpg` on the left
- `/farmer/come-in` — should show `dawn-pasture.jpg` on the left
- `/farmer/sign-up`, `/farmer/forgot-password`, `/farmer/reset-password`
  — all show `dawn-pasture.jpg`

If you see the SVG (warm sky over rolling hills with a sun), the file
isn't being served — check the file name, the path, and the env var.

To swap back to SVG temporarily, just unset `NEXT_PUBLIC_USE_WATERCOLORS`
and rebuild.
